프로젝트 생성 + DB 준비

☕ Spring Boot 프로젝트 생성 첫 번째
STS → File → New → Spring Starter Project

Dependencies: Spring Web, Spring Data JPA, MySQL Driver
Group: com.sa.trk   Artifact: sa-tracker

설정 클래스 (Config)
반나절
RestTemplate(외부 API 호출 도구)과 CORS(React 허용) 설정.
딱 2개 파일만 만들면 됩니다. 한번 만들면 거의 안 건드립니다.
🔧 config/NexonApiConfig.java 먼저
RestTemplate을 Bean으로 등록하는 설정 클래스
package com.sa.trk.config; import org.springframework.context.annotation.Bean; import org.springframework.context.annotation.Configuration; import org.springframework.web.client.RestTemplate; @Configuration public class NexonApiConfig { @Bean public RestTemplate restTemplate() { return new RestTemplate(); } }
🌐 config/WebConfig.java 다음
React 개발 서버(localhost:5173)에서의 API 호출을 허용하는 CORS 설정
package com.sa.trk.config; import org.springframework.context.annotation.Configuration; import org.springframework.web.servlet.config.annotation.*; @Configuration public class WebConfig implements WebMvcConfigurer { @Override public void addCorsMappings(CorsRegistry registry) { registry.addMapping("/api/**") .allowedOrigins("http://localhost:5173") .allowedMethods("GET") .allowedHeaders("*"); } }

2
Entity + Repository
반나절
DB 테이블과 연결되는 Java 클래스. clan_members 테이블의 Java 버전이라고 생각하면 됩니다.
Repository는 interface만 만들면 Spring이 알아서 SQL을 생성해줍니다.
📦 entity/ClanMember.java 먼저
@Entity — DB 테이블과 1:1 매핑되는 클래스
package com.sa.trk.entity; import jakarta.persistence.*; import java.time.LocalDateTime; @Entity @Table(name = "clan_members") public class ClanMember { @Id @GeneratedValue(strategy = GenerationType.IDENTITY) private Long id; @Column(name = "user_name", nullable = false, unique = true) private String userName; @Column(name = "clan_name") private String clanName = "모"; private String ouid; @Column(name = "created_at") private LocalDateTime createdAt; @Column(name = "updated_at") private LocalDateTime updatedAt; // Getter & Setter (전부 만들어야 함) public Long getId() { return id; } public void setId(Long id) { this.id = id; } public String getUserName() { return userName; } public void setUserName(String userName) { this.userName = userName; } public String getClanName() { return clanName; } public void setClanName(String clanName) { this.clanName = clanName; } public String getOuid() { return ouid; } public void setOuid(String ouid) { this.ouid = ouid; } public LocalDateTime getCreatedAt() { return createdAt; } public LocalDateTime getUpdatedAt() { return updatedAt; } }
🔍 repository/ClanMemberRepository.java 다음
interface만 만들면 끝! 구현 코드 없이 Spring이 알아서 해줌
package com.sa.trk.repository; import com.sa.trk.entity.ClanMember; import org.springframework.data.jpa.repository.JpaRepository; import java.util.Optional; public interface ClanMemberRepository extends JpaRepository<ClanMember, Long> { // 닉네임으로 클랜원 찾기 (Spring이 자동 생성) Optional<ClanMember> findByUserName(String userName); }
✅ 확인 방법
서버 재시작 → 콘솔에 "Hibernate: select..." 같은 SQL 로그가 찍히면 DB 연결 성공!

package com.sa.trk.dto.player; public class OuidResponseDto { private String ouid; public String getOuid() { return ouid; } public void setOuid(String ouid) { this.ouid = ouid; } }

4
Service 계층 (가장 중요!)
3~4일
핵심 실제로 NEXON API를 호출하고 데이터를 조합하는 두뇌 역할.
이 단계가 제일 오래 걸리고 제일 중요합니다. 한 메서드씩 만들고 테스트하세요!
⭐ service/NexonApiService.java 가장 먼저!
NEXON API를 직접 호출하는 서비스. getOuid()부터 하나씩 만들어가세요.
package com.sa.trk.service; import com.sa.trk.dto.player.*; import org.springframework.beans.factory.annotation.Value; import org.springframework.http.*; import org.springframework.stereotype.Service; import org.springframework.web.client.RestTemplate; import java.net.URLEncoder; import java.nio.charset.StandardCharsets; @Service public class NexonApiService { private final RestTemplate restTemplate; private static final String BASE = "https://open.api.nexon.com"; @Value("${nexon.api.key}") private String apiKey; public NexonApiService(RestTemplate restTemplate) { this.restTemplate = restTemplate; } // 공통: API KEY를 헤더에 담는 메서드 private HttpHeaders headers() { HttpHeaders h = new HttpHeaders(); h.set("x-nxopen-api-key", apiKey); return h; } // ★ 1번째로 만들 메서드: 닉네임 → ouid public String getOuid(String userName) { String url = BASE + "/suddenattack/v1/id?user_name=" + URLEncoder.encode(userName, StandardCharsets.UTF_8); HttpEntity<String> entity = new HttpEntity<>(headers()); ResponseEntity<OuidResponseDto> res = restTemplate.exchange(url, HttpMethod.GET, entity, OuidResponseDto.class); return res.getBody().getOuid(); } // ★ 2번째: ouid → 기본 정보 public UserBasicDto getUserBasic(String ouid) { String url = BASE + "/suddenattack/v1/user/basic?ouid=" + ouid; HttpEntity<String> entity = new HttpEntity<>(headers()); return restTemplate.exchange(url, HttpMethod.GET, entity, UserBasicDto.class) .getBody(); } // ★ 같은 패턴으로 getUserRank, getUserTier, getRecentInfo, getMatches, getMatchDetail 추가 // ... (url과 반환 타입만 바꾸면 됨) }
💾 service/ClanService.java 두 번째
ouid를 DB에 캐싱하는 핵심 로직. API 호출 횟수를 줄여줍니다.
@Service public class ClanService { private final ClanMemberRepository repo; private final NexonApiService nexonApi; public ClanService(ClanMemberRepository repo, NexonApiService nexonApi) { this.repo = repo; this.nexonApi = nexonApi; } // DB에 ouid 있으면 재사용, 없으면 API 호출 후 저장 public String getOrFetchOuid(String userName) { ClanMember member = repo.findByUserName(userName).orElse(null); // DB에 ouid가 이미 있으면 바로 반환 (API 호출 안 함!) if (member != null && member.getOuid() != null) { return member.getOuid(); } // 없으면 API에서 가져와서 DB에 저장 String ouid = nexonApi.getOuid(userName); if (member != null) { member.setOuid(ouid); repo.save(member); } return ouid; } // 클랜원 전체 목록 public List<ClanMember> getAllMembers() { return repo.findAll(); } }
🔗 service/PlayerService.java 세 번째
여러 API 결과를 하나로 합치는 조합기. PlayerPageDto를 만들어서 반환.
@Service public class PlayerService { private final ClanService clanService; private final NexonApiService nexonApi; // ... 생성자 public PlayerPageDto getPlayerPage(String name) { String ouid = clanService.getOrFetchOuid(name); PlayerPageDto dto = new PlayerPageDto(); dto.setBasic(nexonApi.getUserBasic(ouid)); dto.setRank(nexonApi.getUserRank(ouid)); dto.setTier(nexonApi.getUserTier(ouid)); dto.setRecent(nexonApi.getRecentInfo(ouid)); dto.setMatches(nexonApi.getMatches(ouid, null, null)); return dto; // 이게 통째로 JSON이 됨! } }

⚠️ 주의: NexonApiService의 getOuid()를 먼저 만들고 반드시 테스트하세요! 이게 돌아야 나머지도 돌아갑니다. 임시 TestController를 만들어서 브라우저에서 확인하는 것을 강력 추천합니다.

✅ 확인 방법 — 임시 테스트 컨트롤러
// 테스트용으로 임시로 만들고, 나중에 삭제해도 됨 @RestController public class TestController { @Autowired NexonApiService api; @GetMapping("/test/ouid") public String test(@RequestParam String name) { return api.getOuid(name); // 브라우저에서 /test/ouid?name=닉네임 } } → 브라우저에 ouid 문자열이 뜨면 성공!

💡 개발 순서 팁: getOuid 성공 → getUserBasic 추가 → getUserRank 추가 → ... 이렇게 한 메서드씩 늘려가세요. 한꺼번에 다 만들면 에러 찾기가 어려워요.

@RestController 만들기
1일
React가 호출할 /api/... 엔드포인트. Service를 호출하고 결과를 return만 하면 됩니다.
Controller는 최대한 얇게! 로직을 여기에 쓰면 안 됩니다.
🎯 controller/PlayerApiController.java
package com.sa.trk.controller; import com.sa.trk.dto.player.PlayerPageDto; import com.sa.trk.service.PlayerService; import org.springframework.web.bind.annotation.*; @RestController @RequestMapping("/api") public class PlayerApiController { private final PlayerService playerService; public PlayerApiController(PlayerService playerService) { this.playerService = playerService; } // React에서: axios.get('/api/player?name=닉네임') @GetMapping("/player") public PlayerPageDto getPlayer(@RequestParam String name) { return playerService.getPlayerPage(name); // ↑ 이게 전부! JSON으로 자동 변환됨 } }
🎯 controller/MatchApiController.java
@RestController @RequestMapping("/api") public class MatchApiController { private final NexonApiService nexonApi; // ... 생성자 @GetMapping("/match/{matchId}") public MatchDetailDto getMatchDetail(@PathVariable String matchId) { return nexonApi.getMatchDetail(matchId); } }
🎯 controller/ClanApiController.java
@RestController @RequestMapping("/api") public class ClanApiController { private final ClanService clanService; // ... 생성자 @GetMapping("/clan/members") public List<ClanMemberDto> getMembers() { return clanService.getAllMembers().stream() .map(m -> new ClanMemberDto(m.getUserName(), m.getClanName())) .toList(); } }
