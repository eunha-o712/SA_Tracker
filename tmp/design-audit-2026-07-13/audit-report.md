# SATrk 디자인·UX 통합 검수

- 검수일: 2026-07-13
- 환경: localhost:5173, 비로그인 및 로컬 검수 계정, 데스크톱 1280×720 / 모바일 390×844
- 범위: 홈, 프로필 검색/결과, 웨폰, 매치, 랭킹, 로그인, 마이페이지, 클랜 검색/결과, 접근 제어, 푸터 정책 링크
- Figma 보드: https://www.figma.com/design/LYqyrtaYPyFAlJtnDbEaiV

## 구현 상태

- 완료: 개인정보처리방침 페이지와 라우트
- 완료: 인트로 2.5초 조기 진입, 건너뛰기, 모션 감소 환경 생략
- 완료: 모바일 헤더 높이와 로그인 버튼 가독성 개선
- 완료: 프로필 빈 상태 여백 축소와 `RECORD ROOM` 오탈자 수정
- 완료: 긴 프로필용 섹션 바로가기
- 완료: 검색·로그인·프로필 주요 행동 문구 한글화
- 완료: 로그인 상태 모바일 헤더 겹침 해소와 내비게이션 터치 영역 확대
- 완료: 마이페이지·클랜 로스터·즐겨찾기 행동 문구 한글화

## 전체 판단

출시 후보 수준의 완성도다. 검정·네온 그린·전술 HUD 이미지가 모든 핵심 페이지에서 일관되고, 현재 메뉴 표시와 섹션 카드 구조도 명확하다. 최초 검수에서 확인된 개인정보처리방침, 인트로 대기, 모바일 헤더, 프로필 탐색 문제는 수정됐다. 남은 과제는 실제 기기에서의 키보드·스크린리더·색 대비 수치 검증과 네트워크 오류 상태 점검이다.

## 단계별 상태

1. 홈 — 양호: 브랜드와 검색 행동이 첫 화면에서 명확하다.
2. 프로필 검색 전 — 양호: `RECORD ROOM` 표기와 모바일 빈 상태 여백을 수정했다.
3. 프로필 결과 — 양호: 기본 정보에서 상세 분석으로 내려가는 구조가 자연스럽다. 다만 전체 길이가 매우 길다.
4. 웨폰 — 양호: 별도 배너와 검색 목적이 프로필과 구분된다.
5. 매치 — 보통: 기능은 풍부하지만 모바일 첫 화면에 검색·빠른 탭·상세 필터가 연속되어 밀도가 높다.
6. 랭킹 — 양호: 목적과 검색 행동이 명확하며 웨폰 페이지와 구조적으로 일관된다.
7. 로그인/마이페이지 — 양호: 로그인 후 닉네임이 헤더와 프로필 기본값에 반영되고, 마이페이지 정보 및 행동이 정상 표시된다.
8. 클랜 — 양호: 비로그인 접근 제어, 로그인 후 빈 상태, 닉네임 검색, 결과 요약·로스터·전력 순위가 정상 동작한다.
9. 개인정보처리방침 — 양호: `/privacy`가 실제 정책 문서 화면으로 열린다.

## 우선순위 발견 사항

### 해결됨 — 개인정보처리방침 링크가 빈 화면

푸터에 링크가 항상 노출되지만 `/privacy` 라우트가 없어 아무 UI도 표시되지 않는다. 신뢰와 출시 준비 측면에서 가장 먼저 수정해야 한다.

증거: `12-privacy-broken-mobile.png`

### 해결됨 — 모바일 헤더의 정보가 지나치게 작음

390px 화면에서 로고와 LOGIN이 사실상 읽기 어려운 크기이며, 메뉴 글자도 작다. 메뉴 자체는 한 줄 안에 들어오지만 확대 없이 읽는 사용자에게 부담이 있다. 터치 영역·키보드 포커스·색 대비는 별도 측정이 필요하다.

증거: `10-profile-search-mobile.png`, `11-matches-mobile.png`

### 해결됨 — 첫 방문마다 7.5초 인트로를 끝까지 봐야 함

영상 길이는 약 7.5초이며 건너뛰기 수단이 없다. 로딩을 숨기는 장점은 있지만, 새 창이나 새 세션마다 검색까지의 시간이 길어진다. 데이터가 먼저 준비되면 2~3초 이후 건너뛰기 또는 즉시 진입을 허용하는 편이 안전하다. `prefers-reduced-motion` 사용자에게는 영상 생략도 필요하다.

### 해결됨 — 모바일 프로필 빈 상태의 수직 여백이 큼

빈 상태 카드 아래에서 푸터까지 큰 검정 공백이 생겨 화면이 덜 완성된 것처럼 보인다. 빈 상태가 짧은 페이지에서는 콘텐츠 높이를 줄이거나 안내 카드의 세로 균형을 조정하는 것이 좋다.

증거: `10-profile-search-mobile.png`

### 해결됨 — 프로필 결과가 한 페이지에 너무 길게 이어짐

1280×720 기준 문서 높이가 약 4,700px이다. 기본정보, 랭킹, 웨폰, 맵, K/D/A, 폼, 성향, AI 분석이 모두 직렬 배치되어 후반 분석을 찾기 어렵다. 상단 섹션 바로가기나 분석 그룹 접기 기능이 있으면 탐색 부담이 줄어든다.

증거: `07-profile-loaded-desktop.png`, `08-profile-analysis-desktop.png`

### 대부분 해결됨 — 화면 간 문구와 검색 컴포넌트가 조금씩 다름

프로필은 `PLAYER SEARCH / SEARCH`, 웨폰은 `닉네임 / 조회`, 매치는 `닉네임 / 검색`, 랭킹은 `닉네임 / 검색`을 쓴다. 로그인 화면은 거의 전부 영어이고 결과 화면에는 `RANK Summ.`, `Day of First Shot`, `ADD FAVORITE`처럼 한영 혼용이 남아 있다. 브랜드용 영어 제목은 유지하되 행동 문구와 설명문은 한 언어로 맞추는 편이 자연스럽다.

증거: `02-profile-search-desktop.png`~`06-login-desktop.png`

### 해결됨 — `RECODE ROOM` 표기 확인 필요

다른 화면과 배너는 `RECORD ROOM`을 사용하지만 프로필 빈 상태만 `RECODE ROOM`이다. 의도된 이름이 아니라면 오탈자다.

## 강점

- 모든 주요 페이지의 아트워크와 색상 체계가 매우 일관적이다.
- 선택된 상단 메뉴가 즉시 구분된다.
- 검색 전 빈 상태가 목적과 다음 행동을 설명한다.
- 프로필 분석 카드는 수치 위계와 그룹 구분이 명확하다.
- 비로그인 상태의 클랜 접근 제어가 예상대로 동작한다.
- 로그인 후 헤더에서 내 닉네임과 마이페이지 진입점이 바로 드러난다.
- 클랜 결과는 개요 → 로스터 → 전력 순위 → 최근 경기 순서가 자연스럽고 모바일에서 2열 카드로 안정적으로 재배치된다.

## 로그인 상태 추가 검수

1. 마이페이지 데스크톱 — 양호: 계정 정보 4개와 두 가지 관리 행동의 위계가 명확하다.
2. 마이페이지 모바일 — 양호: 단일 열 카드로 재배치되고 긴 이메일/OUID가 영역을 넘지 않는다.
3. 클랜 검색 전 — 양호: 검색 목적, 입력창, 빈 상태 안내가 분리되어 다음 행동을 이해하기 쉽다.
4. 클랜 검색 결과 — 양호: 클랜명과 핵심 지표가 첫 결과 구간에서 바로 보인다.
5. 클랜 로스터·전력 순위 — 양호: 등록 전 빈 상태와 집계 기준이 명확하다.
6. 로그아웃 — 양호: 홈으로 이동하고 헤더·클랜 링크가 비로그인 상태로 즉시 전환된다.

## 검증 한계

- 로컬 검수 전용 계정으로 로그인 흐름을 확인했으며 실제 사용자 계정·운영 데이터의 권한 정책은 확인하지 않았다.
- 스크린샷으로 색 대비 수치, 스크린리더 낭독, 전체 키보드 순서, 실제 터치 정확도를 확정할 수 없다.
- API 지연과 오류 상태는 정상 응답 위주로 확인했으며 모든 네트워크 실패 상태를 재현하지 않았다.
